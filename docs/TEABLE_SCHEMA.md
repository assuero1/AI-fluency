# Teable Schema Map

This document maps product entities to Teable tables. Field names may be adapted to Teable conventions during implementation, but the meaning should stay stable.

## Table: Users

Purpose: stores the user profile.

Fields:

- `id`: record id
- `name`: text
- `avatar_url`: url/text
- `active_language_id`: relation to LanguageProfiles
- `timezone`: text
- `created_at`: date

## Table: LanguageProfiles

Purpose: one learning profile per language.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `language_code`: text
- `language_name`: text
- `level`: single select
- `learning_goal`: long text
- `correction_style`: single select
- `audio_enabled`: checkbox
- `transcript_enabled`: checkbox
- `calendar_memory_enabled`: checkbox
- `weekly_conversation_goal`: number
- `weekly_word_goal`: number
- `created_at`: date
- `updated_at`: date

## Table: AIProviderSettings

Purpose: stores AI provider/model status. Real secrets may be env-only instead of Teable records.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `provider`: single select
- `base_url`: url/text
- `api_key_masked`: text
- `chat_model`: text
- `reasoning_model`: text
- `temperature`: number
- `max_tokens`: number
- `is_active`: checkbox
- `last_test_status`: single select
- `last_test_at`: date
- `created_at`: date
- `updated_at`: date

Security:

- Prefer storing only masked labels/status in Teable.
- Store actual secrets in server environment variables or encrypted server-side storage.

## Table: VoiceProviderSettings

Purpose: stores Kokoro configuration/status.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `provider`: single select, default `kokoro`
- `base_url`: url/text
- `api_key_masked`: text
- `default_voice`: text
- `speech_speed`: number
- `output_format`: single select
- `is_active`: checkbox
- `last_test_status`: single select
- `last_test_at`: date
- `created_at`: date
- `updated_at`: date

## Table: Conversations

Purpose: one study session.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `language_profile_id`: relation to LanguageProfiles
- `topic_id`: relation to Topics
- `mode`: single select
- `status`: single select
- `started_at`: date
- `ended_at`: date
- `duration_seconds`: number
- `ai_model_used`: text
- `summary`: long text

Modes:

- `free_conversation`
- `suggested_topic`
- `custom_topic`
- `review_words`
- `calendar_focus`

## Table: Messages

Purpose: every user or assistant message.

Fields:

- `id`: record id
- `conversation_id`: relation to Conversations
- `role`: single select
- `text`: long text
- `audio_url`: url/text
- `transcript_text`: long text
- `language_detected`: text
- `tokens_used`: number
- `created_at`: date

Roles:

- `user`
- `assistant`
- `system`

## Table: Corrections

Purpose: language corrections generated by AI.

Fields:

- `id`: record id
- `conversation_id`: relation to Conversations
- `message_id`: relation to Messages
- `original_text`: long text
- `corrected_text`: long text
- `error_type`: single select
- `explanation`: long text
- `severity`: single select
- `should_interrupt`: checkbox
- `created_at`: date

Error types:

- `grammar`
- `vocabulary`
- `pronunciation`
- `tense`
- `preposition`
- `word_order`
- `naturalness`
- `spelling`

## Table: Words

Purpose: consolidated user vocabulary.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `language_profile_id`: relation to LanguageProfiles
- `lemma`: text
- `display_text`: text
- `translation`: text
- `part_of_speech`: text/single select
- `familiarity_score`: number
- `total_uses`: number
- `last_used_at`: date
- `first_used_at`: date
- `review_due_at`: date
- `review_interval_days`: number
- `review_ease`: number, bounded by the versioned SRS algorithm
- `review_streak`, `lapse_count`: number
- `last_reviewed_at`: date
- `last_rating`: single select (`forgot`, `hard`, `good`, `easy`)
- `average_response_time_ms`: number
- `review_state`: single select (`new`, `learning`, `review`, `difficult`, `suspended`)
- `review_version`: text

## Table: WordOccurrences

Purpose: every contextual use of a word.

Fields:

- `id`: record id
- `word_id`: relation to Words
- `conversation_id`: relation to Conversations
- `message_id`: relation to Messages
- `used_text`: text
- `sentence_context`: long text
- `was_correct`: checkbox
- `created_at`: date

## Table: DailyFeedbacks

Purpose: daily learning memory.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `language_profile_id`: relation to LanguageProfiles
- `date`: date
- `strengths`: long text
- `weaknesses`: long text
- `recommended_focus`: long text
- `recurring_errors`: long text/json
- `new_words_count`: number
- `correction_score`: number
- `fluency_score`: number
- `suggested_topics`: long text/json
- `created_at`: date

## Table: Topics

Purpose: user-created or AI-generated conversation themes.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `language_profile_id`: relation to LanguageProfiles
- `title`: text
- `source`: single select
- `reason`: long text
- `related_feedback_id`: relation to DailyFeedbacks
- `related_words`: relation to Words, multiple
- `difficulty`: single select
- `created_at`: date

Sources:

- `user_custom`
- `ai_suggestion`
- `calendar_based`
- `weak_words`
- `recurring_error`

## Table: PracticeSessions

Purpose: grouped practice actions, especially word review/focus practice.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `language_profile_id`: relation to LanguageProfiles
- `conversation_id`: relation to Conversations
- `type`: single select
- `focus`: text
- `status`: preparing, active, completed, abandoned or failed
- `started_at`, `ended_at`, `duration_seconds`
- `criterion`, `requested_word_count`, `selected_word_count`
- `unique_card_count`, `presentation_count`
- `correct_count`, `incorrect_count`, `score`
- `language_code`, `configuration_json`, `parent_session_id`
- `created_at`: date
- `updated_at`: date

## Table: Flashcards

Purpose: immutable card snapshot for one practice session.

Fields:

- `practice_session_id`, `target_word_id`, `supporting_word_ids`
- `card_type`, `prompt`, `expected_answer`, `accepted_answers`
- `translation`, `explanation`, `sentence`, `audio_text`
- `difficulty`, `initial_position`, `generation_source`, `created_at`

## Table: FlashcardAttempts

Purpose: audit every card presentation and learner answer.

Fields:

- `practice_session_id`, `flashcard_id`, `word_id`
- `presentation_number`, `client_attempt_id`
- `user_answer`, `normalized_answer`, `match_result`
- `suggested_rating`, `final_rating`, `was_correct`
- `response_time_ms`, `used_speech`, `audio_replay_count`
- `used_slow_audio`, `answered_after_audio_replay`, `audio_failed`, `created_at`

## Table: AppEvents

Purpose: product/debug event trail.

Fields:

- `id`: record id
- `user_id`: relation to Users
- `event_name`: text
- `payload`: long text/json
- `created_at`: date

Initial events:

- `onboarding_completed`
- `teable_connected`
- `ai_provider_connected`
- `kokoro_connected`
- `topic_suggested`
- `conversation_started`
- `correction_shown`
- `word_saved`
- `daily_feedback_created`
